Complete the AI Helper

~15 minutes • TODOs 1–2
File: utils/ai_helper.py
What you're building:

The AI helper is the "engine" of your toolkit. It contains two functions that all three tools share: one for sending a single question to Claude, and one for having a back-and-forth conversation. Once this file works, every tool will be able to talk to AI.

1.1 — Open the File

In VS Code, open the file utils/ai_helper.py. You can find it in the sidebar file explorer, or press Ctrl+P (Windows) / Cmd+P (Mac) and type ai_helper.py.

Read through the file. Notice it already has two functions set up, but each one ends with pass — that's Python's way of saying "do nothing." Your job is to replace those pass lines with real code.

1.2 — Complete TODO 1: Single Message API Call

Find the comment that says # TODO 1: Call the Claude API. You need to replace the pass line with code that:

  1. Calls client.messages.create() with the right parameters
  2. Returns the text from Claude's response

Understanding the Parameters

  • model — Which AI model to use. We're using "claude-sonnet-4-6", a fast and capable model.
  • max_tokens — The maximum length of Claude's response (2048 tokens is roughly 1,500 words).
  • system — The "system prompt" that tells Claude how to behave (like giving it a job description).
  • messages — The actual question or content you're sending.

The Code to Write

Replace the pass at the end of the get_ai_response function with this:

utils/ai_helper.py — TODO 1 
    # Call the Claude API
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        system=system_prompt,
        messages=[{"role": "user", "content": user_message}]
    )

    # Return just the text from Claude's response
    return response.content[0].text
Make sure the indentation is correct!

The code should be indented with 4 spaces, aligned with the existing comments inside the function. Python is very picky about indentation.

1.3 — Complete TODO 2: Conversation API Call

Find the comment that says # TODO 2: Call the Claude API with a conversation. This is almost identical to TODO 1, except instead of sending a single message, you send the entire conversation history.

Replace the pass at the end of the get_chat_response function with this:

utils/ai_helper.py — TODO 2 
    # Call the Claude API with the full conversation
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        system=system_prompt,
        messages=messages
    )

    # Return just the text from Claude's response
    return response.content[0].text
What's the difference?

In TODO 1, messages is a list with just one item (the user's question). In TODO 2, messages is a list of the entire conversation — alternating between user messages and assistant responses. This is what lets the Socratic Tutor remember what was said earlier in the conversation.

1.4 — Verify It Works

Let's run a quick test to make sure your AI helper is working. Create a temporary test file (you can delete it afterward):

If everything is working, you should see Claude's 5-word greeting printed in the terminal. If you get an error, check the troubleshooting section below.

Troubleshooting: AI Helper Issues

"AuthenticationError" or "Invalid API key"

Your API key is not set correctly. Re-run the export or set command from Step 0.5. Make sure you're in the same terminal window where you set the key.

"ModuleNotFoundError: No module named 'anthropic'"

Make sure your virtual environment is activated (you should see (venv) in your prompt). Then run pip install anthropic.

"IndentationError"

Python requires consistent indentation. Make sure your code uses 4 spaces (not tabs). In VS Code, look at the bottom-right corner — it should say "Spaces: 4".

"SyntaxError"

Double-check that you've copied the code exactly, including all parentheses, brackets, and quotes. Make sure you deleted the pass line.

What you're building:

A tool where you paste a student's lab report and get detailed, rubric-based feedback from AI. This is your first hands-on experience with prompt engineering — the art of writing instructions that get the AI to behave exactly how you want.

2.1 — Open the File

Open pages/1_📝_Lab_Report_Feedback.py in VS Code.

Scroll down and find the two TODO sections. TODO 3 is about writing the system prompt, and TODO 4 is about building the user interface.

2.2 — Complete TODO 3: Write the System Prompt

The system prompt is the single most important part. It's like writing a detailed job description for an AI teaching assistant. A good system prompt produces excellent feedback; a vague one produces generic, unhelpful responses.

Good Prompts vs. Bad Prompts

Weak prompt:

"Give feedback on a lab report."

Strong prompt:

"You are a supportive high school science teacher. Evaluate on 4 criteria, score each out of 25, start with strengths, use growth-mindset language..."

Find the line that says system_prompt = "" and replace it with:

pages/1_📝_Lab_Report_Feedback.py — TODO 3 
system_prompt = """You are a supportive and experienced high school science teacher
reviewing a student's lab report. Your feedback should help the student
grow as a scientist.

EVALUATION RUBRIC (score each category out of 25 points):

1. HYPOTHESIS (0-25 points)
   - Is it clearly stated?
   - Is it testable and specific?
   - Does it identify the independent and dependent variables?

2. METHODOLOGY (0-25 points)
   - Is the procedure detailed enough to be reproducible?
   - Are variables properly controlled?
   - Is the sample size appropriate?
   - Are safety considerations mentioned?

3. DATA ANALYSIS (0-25 points)
   - Is the data presented clearly (tables, graphs)?
   - Are calculations accurate with correct units?
   - Are trends or patterns identified?

4. CONCLUSION (0-25 points)
   - Does it directly address the hypothesis?
   - Is it supported by the data?
   - Are sources of error discussed?
   - Are future improvements suggested?

FORMAT YOUR FEEDBACK LIKE THIS:
- Start with 2-3 things the student did WELL (be specific!)
- Give the score and detailed feedback for each rubric category
- End with 2-3 specific, actionable "Next Steps" for improvement
- Add a total score out of 100

TONE: Encouraging but honest. Use growth-mindset language like
"You're on the right track..." or "To strengthen this section, try..."
Never be harsh or discouraging. Remember these are high school students
who are still learning the scientific method."""
Feel free to customize!

This is your rubric. You might want to weight categories differently, add criteria for your specific subject, or adjust the tone. Experiment and see how changes to the system prompt affect the AI's feedback.

  • st.spinner() — Shows a "loading" animation while the AI is thinking.
  • st.markdown() — Displays formatted text (Claude's response comes back with formatting).

Build the Socratic Tutor

~30 minutes • TODOs 7–8
File: pages/3_🤔_Socratic_Tutor.py
What you're building:

A chat-based AI tutor that uses the Socratic method — it guides students through problem-solving by asking questions, never giving direct answers. This is the most sophisticated tool: it maintains a conversation history so the AI remembers what was said earlier, and the prompt engineering is the most challenging because you have to instruct an AI to not do what it naturally wants to do (give answers).

4.1 — A Quick Word About the Socratic Method

The Socratic method is a teaching approach where the teacher asks a series of guiding questions to help students arrive at understanding on their own. Instead of saying "The answer is 42," a Socratic tutor would ask "What happens when you multiply 6 by 7?"

This is a perfect use case for AI because:

  • The AI can be infinitely patient
  • It can adapt its questions to the student's level
  • It can work with each student one-on-one, even when you're helping other students

4.2 — Open the File

Open pages/3_🤔_Socratic_Tutor.py in VS Code.

4.3 — Complete TODO 7: Write the System Prompt

This is the most important prompt in the entire toolkit. The challenge: Claude naturally wants to be helpful by giving direct answers. You need to override that instinct and tell it to only ask guiding questions.

Find the line system_prompt = "" and replace it with:

pages/3_🤔_Socratic_Tutor.py — TODO 7 
system_prompt = """You are a Socratic tutor for high school STEM students.
Your goal is to help students discover answers through guided questioning,
never by telling them the answer directly.

GOLDEN RULES (follow these strictly):
1. NEVER give the student the direct answer to their question.
2. Instead, ask ONE guiding question that helps them take the next step.
3. If they're stuck, break the problem into a smaller, more manageable piece.
4. If they make an error, don't say "that's wrong." Instead, ask a question
   that helps them discover the mistake themselves.
   Example: "Interesting thought! What would happen if we tested that
   with the number 0? Does your rule still work?"
5. Celebrate correct reasoning enthusiastically:
   "Excellent thinking! You've got it. Now, can we take it one step further?"
6. Use analogies from everyday life to make abstract concepts concrete.
   Example: "Think of electrons like students in a school — they fill up
   the front-row seats (lower energy levels) before moving to the back."
7. Keep responses concise: 2-4 sentences plus one guiding question.
8. ALWAYS end your response with a question to keep the dialogue moving.

CONVERSATION FLOW:
- First message: Greet the student warmly, ask them to describe the
  problem or concept they need help with.
- During the conversation: Guide step-by-step with questions.
- If the student gets the right answer: Celebrate and ask if they
  want to try a harder variation or a different problem.

TONE: Warm, patient, encouraging, never condescending. You're a mentor,
not a lecturer. Use casual but respectful language appropriate for
high school students."""

4.4 — Complete TODO 8: Build the Chat Interface

This is different from the other two pages because instead of a simple form, you're building a chat interface. Streamlit has built-in chat components that make this surprisingly easy.

Understanding Session State

When a user interacts with a Streamlit app, the entire script reruns from top to bottom. That means variables would normally be lost between interactions! st.session_state solves this — it's a persistent dictionary that survives across reruns. We use it to store the conversation history.

Find the comment # Your chat code goes here! and replace it with:

pages/3_🤔_Socratic_Tutor.py — TODO 8 
# ── Sidebar settings ──
with st.sidebar:
    subject = st.selectbox("Subject", ["Biology", "Chemistry", "Physics", "Math", "Engineering"])
    grade = st.selectbox("Grade Level", ["9th", "10th", "11th", "12th", "AP"])
    if st.button("🗑️ Clear Chat"):
        st.session_state.messages = []
        st.rerun()

# ── Initialize chat history ──
if "messages" not in st.session_state:
    st.session_state.messages = []

# ── Display existing chat messages ──
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# ── Handle new user input ──
if prompt := st.chat_input("Ask a question or describe a problem..."):
    # Add user message to history and display it
    st.session_state.messages.append({"role": "user", "content": prompt})
    with st.chat_message("user"):
        st.markdown(prompt)

    # Get AI response
    with st.chat_message("assistant"):
        with st.spinner("Thinking..."):
            full_system = f"{system_prompt}\n\nYou are tutoring a {grade} grade {subject} student."
            response = get_chat_response(full_system, st.session_state.messages)
        if response:
            st.markdown(response)
            st.session_state.messages.append({"role": "assistant", "content": response})
        else:
            st.error("Could not get a response. Check your API key.")

How the Chat Loop Works

  1. Initialize: st.session_state.messages starts as an empty list.
  2. Display: The for loop shows all previous messages (so they don't disappear when the page reruns).
  3. Input: st.chat_input() shows a text box at the bottom of the page, just like a messaging app.
  4. Process: When the user sends a message, it gets added to the history, sent to Claude along with the full history, and Claude's response gets added to the history too.
  5. Repeat: The page reruns, displaying all messages including the new ones.
Why := (walrus operator)?

The line if prompt := st.chat_input(...) uses the "walrus operator." It's a shortcut that assigns the value and checks if it's not empty, all in one line. It's equivalent to writing prompt = st.chat_input(...) followed by if prompt:.

Troubleshooting: Socratic Tutor Issues

The tutor gives direct answers instead of asking questions

Your system prompt might not be strong enough. Make sure it includes phrases like "NEVER give the direct answer" and "ALWAYS end with a question." The stronger and more explicit your instructions, the better the AI follows them.

Chat history disappears when I change the subject

This is expected! Changing a sidebar widget causes Streamlit to rerun. The messages are still in session state, but the new subject will be applied to future responses. Use "Clear Chat" to start a conversation in the new subject.

"get_chat_response is not defined"

Make sure the import at the top of the file says from utils.ai_helper import get_chat_response (not get_ai_response). This page uses the chat version!

The chat input box doesn't appear

st.chat_input() always appears at the bottom of the page. If your code has an error above it, the input might not render. Check the terminal for error messages.

Troubleshooting: Deployment Issues

"ModuleNotFoundError" in deployment logs

Make sure your requirements.txt file is in the root of the repository and includes both streamlit and anthropic.

App crashes with "API key" error

Check that you added the ANTHROPIC_API_KEY in the Advanced settings → Secrets section. Make sure the key is valid and has not expired.

The app deploys but pages are missing

Make sure the pages/ folder and all its files are committed and pushed to GitHub. Run git status to check.
