Exercise 1
"Hello, Python!" — Your First Script
~10 min
📚 What You'll Learn

How to create a Python file, write your first line of code, and run it in the terminal.

  1. Open VS Code on your computer.
  2. Create a new file and save it as hello.py
  3. Type the following code into your file:
Python 
print("Hello, CSPD 2026!")
  1. Open the terminal in VS Code by pressing Ctrl+` (backtick) or going to View → Terminal.
  2. Run your script by typing the following command:
Terminal 
python hello.py
Tip — Mac Users

On Mac, you may need to use python3 hello.py instead of python hello.py.

Expected Output: Hello, CSPD 2026!
  1. Celebrate! 🎉 You just wrote and ran your first Python program!
🏆 Mini-Challenge

Change the message to include your name and the subject you teach. For example: "Hello! I'm Ms. Chen and I teach Chemistry!"

Troubleshooting
  • "python is not recognized" — Try python3 instead, or check that Python is added to your system PATH.
  • SyntaxError — Make sure you have matching quotation marks and parentheses.
  • File not found — Make sure your terminal is in the same folder where you saved hello.py. Use cd to navigate there.
Exercise 2
"Variables & Building Blocks"
~10 min
📚 What You'll Learn

How to store information in variables, format text with f-strings, and organize data in lists and dictionaries.

  1. Create a new file called variables.py
  2. Type the following code:
Python 
# My first variables
teacher_name = "Ms. Rodriguez"
subject = "Biology"
grade = 10

# Using an f-string to build a message
message = f"Hi! I'm {teacher_name}, and I teach {subject} to grade {grade} students."
print(message)
Expected Output: Hi! I'm Ms. Rodriguez, and I teach Biology to grade 10 students.
Key Concepts
  • Variables — named containers that store data (like labeled boxes).
  • Strings — text wrapped in quotes: "Hello"
  • f-strings — strings that start with f and let you embed variables inside {curly braces}.
  • Numbers — integers (whole numbers like 10) don't need quotes.

Lists and Dictionaries

Python also lets you organize collections of data. Add this to your file:

Python 
# A list is an ordered collection (like a bulleted list)
topics = ["photosynthesis", "cell division", "genetics"]
print("My topics:", topics)

# A dictionary maps labels to values (like a rubric!)
rubric = {
    "methodology": 25,
    "data_analysis": 25,
    "conclusion": 25,
    "presentation": 25
}
print("Rubric:", rubric)
Key Concepts
  • Lists — ordered collections in [square brackets], separated by commas.
  • Dictionaries — key-value pairs in {curly braces}. Think of them like a lookup table.
🏆 Mini-Challenge

Add more variables (school name, favorite topic, years teaching) and build a longer introduction using an f-string. Try creating a list of your own topics and a rubric dictionary for your class!

Troubleshooting
  • NameError — Check that variable names are spelled exactly the same wherever you use them (Python is case-sensitive).
  • SyntaxError with f-string — Make sure the string starts with f" (the f must be right before the opening quote).
  • Missing comma in list — Each item in a list must be separated by a comma.
Exercise 3
"Installing Packages — Your AI Toolkit"
~10 min
📚 What You'll Learn

How to install Python packages using pip, and verify they're ready to use.

Key Concept — What are Packages?

Packages are pre-built code libraries that other developers have created so you don't have to write everything from scratch. Think of them like apps on your phone — someone else built them, and you install and use them. pip is the tool that installs packages (like an app store for Python).

  1. Open your terminal in VS Code.
  2. Run the following command to install both packages:
Terminal 
pip install anthropic streamlit
Tip — Mac Users

On Mac, use pip3 install anthropic streamlit instead.

Be Patient!

These are large packages and may take a minute or two to install. You'll see progress messages scrolling in the terminal — that's normal!

  1. Verify the installation by creating a file called check_install.py with this code:
Python 
import anthropic
import streamlit
print("All packages installed successfully! ✅")
  1. Run it: python check_install.py
Expected Output: All packages installed successfully! ✔️
Troubleshooting
  • Permission errors — Add the --user flag: pip install --user anthropic streamlit
  • "pip: command not found" — Try pip3 instead of pip
  • "No module named X" — Make sure you're using the same Python that pip installed to. Try python3 check_install.py if you used pip3.
  • Slow installation — Be patient! Large packages take time, especially on slower connections. As long as the terminal is showing progress, it's working.
  • Red error text during install — Some red/yellow warnings during installation are normal. Only worry if the final line says "ERROR."
Exercise 4
"Your First AI Conversation"
~10 min
📚 What You'll Learn

How to connect to the Claude API and get your first AI-generated response.

  1. First, set up your API key as an environment variable. In your terminal, type:

Mac / Linux:

Terminal 
export ANTHROPIC_API_KEY="your-key-here"

Windows:

Terminal 
set ANTHROPIC_API_KEY=your-key-here
Important

Replace your-key-here with the actual API key provided by your facilitator. Keep this key private — never share it or post it online!

  1. Create a new file called first_ai_call.py
  2. Type the following code:
Python 
import anthropic

# Create a client (it reads your API key from the environment)
client = anthropic.Anthropic()

# Send a message to Claude
message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {"role": "user", "content": "Explain photosynthesis in one paragraph for a 9th grader."}
    ]
)

# Print the response
print(message.content[0].text)
  1. Run it: python first_ai_call.py
  2. Watch as Claude responds with an explanation! 🤖
Expected Output: A paragraph explaining photosynthesis in student-friendly language. (Your exact output will vary — AI responses are generated fresh each time!)
Key Concepts
  • API — Application Programming Interface. A way for your code to "talk to" the AI service over the internet.
  • API Key — Like a password that identifies you to the service.
  • model — Which version of Claude to use.
  • max_tokens — The maximum length of the response (1 token is roughly 1 word).
  • messages — The conversation history you send to Claude.
🏆 Mini-Challenge

Change the question to something from YOUR subject area! For example: "Explain the quadratic formula for a 10th grader" or "Describe Newton's third law using everyday examples."

Troubleshooting
  • AuthenticationError — Your API key is missing or incorrect. Re-run the export or set command with the correct key.
  • "No module named anthropic" — Go back to Exercise 3 and install the package.
  • Connection error — Check your internet connection. The API requires internet access.
  • Rate limit error — Wait a few seconds and try again.
Exercise 5
"Adding a System Prompt — Giving AI a Role"
~10 min
📚 What You'll Learn

How to use system prompts to shape AI behavior and create a specialized teaching assistant.

Key Concept — System Prompts

A system prompt is a set of instructions that tells the AI how to behave. Think of it like giving a substitute teacher your lesson plans and classroom rules before they take over your class. The AI will follow these instructions for the entire conversation.

  1. Create a new file called system_prompt.py
  2. Type the following code:
Python 
import anthropic

client = anthropic.Anthropic()

# Define the AI's personality and behavior
system_prompt = """You are a friendly, patient high school biology teacher.
You explain concepts using simple analogies.
You always encourage students and check for understanding.
When giving feedback, you start with something positive."""

# Send a message with the system prompt
message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system=system_prompt,
    messages=[
        {"role": "user", "content": "Can you check my understanding? I think photosynthesis happens in the mitochondria."}
    ]
)

print(message.content[0].text)
  1. Run it: python system_prompt.py
  2. Notice how the AI responds! It should:
    • Start with something positive (acknowledging the student's effort)
    • Gently correct the misconception
    • Use a simple analogy
    • Check for understanding
Expected Output: A warm, encouraging response that gently corrects the misconception (photosynthesis happens in chloroplasts, not mitochondria), using an analogy and checking for understanding. The exact wording will vary.
Tip

Try running the same question without the system prompt (comment it out by changing system=system_prompt to remove the system parameter). Notice how the response style changes!

🏆 Mini-Challenge

Write a system prompt for YOUR subject and teaching style! Think about:

  • What subject does the AI teach?
  • What tone should it use? (encouraging, humorous, formal?)
  • How should it handle mistakes?
  • What level should explanations be at?
Troubleshooting
  • The AI ignores the system prompt — Make sure the system=system_prompt parameter is included in the messages.create() call.
  • Triple-quote strings — The """ syntax lets you write multi-line strings. Make sure you have three quotes at the start and three at the end.
  • API key issues — If your terminal session restarted, you may need to set the API key again (Exercise 4, Step 1).
Exercise 6
"Hello, Streamlit! — Your First Web App"
~10 min
📚 What You'll Learn

How to create a web application using Streamlit, with interactive input elements.

Key Concept — What is Streamlit?

Streamlit turns your Python scripts into interactive web apps with just a few lines of code. No HTML, CSS, or JavaScript knowledge needed! It's like magic for building quick demos and tools.

  1. Create a new file called my_first_app.py
  2. Type the following code:
Python 
import streamlit as st

st.title("🔬 My STEM AI Assistant")
st.write("Welcome! This is my first web app.")

name = st.text_input("What's your name?")
subject = st.selectbox("What subject do you teach?",
    ["Biology", "Chemistry", "Physics", "Math", "Engineering", "Computer Science", "Other"])

if name:
    st.success(f"Hello, {name}! Let's build something amazing for your {subject} class!")

    topic = st.text_area("Enter a topic you're teaching this week:")
    if st.button("Get a fun fact!"):
        st.info(f"🤖 Imagine an AI generating a fun fact about '{topic}' here... We'll connect the AI in the afternoon!")
  1. Run it with Streamlit (note: this is different from how we ran scripts before!):
Terminal 
streamlit run my_first_app.py
  1. Your browser should open automatically! If not, look for a URL in the terminal (usually http://localhost:8501) and open it manually.
  2. Try interacting with your app — type your name, select a subject, and click the button!
Expected Output: A web page opens in your browser with a title, text input, dropdown, and button. When you type your name, a green success message appears. When you click the button, a blue info message appears.
Streamlit Components You Just Used
  • st.title() — Large title text
  • st.write() — General-purpose text display
  • st.text_input() — Single-line text input
  • st.text_area() — Multi-line text input
  • st.selectbox() — Dropdown menu
  • st.button() — Clickable button
  • st.success() — Green success message
  • st.info() — Blue info message
  • st.warning() — Yellow warning message
Tip

To stop the Streamlit server, go back to your terminal and press Ctrl+C. Every time you save your file, Streamlit will ask if you want to re-run — click "Always rerun" for the best experience!

🏆 Mini-Challenge

Enhance your app! Try adding:

  • A slider for grade level: grade = st.slider("Grade level", 6, 12, 9)
  • More input fields for different types of information
  • A st.warning() message somewhere in your app
  • An st.balloons() call to celebrate (just call it on its own line!)
Troubleshooting
  • "streamlit: command not found" — Try python -m streamlit run my_first_app.py instead.
  • Browser doesn't open — Copy the URL from the terminal (usually http://localhost:8501) and paste it in your browser.
  • Page shows an error — Check the terminal for the full error message. Most often it's a typo in the code.
  • "Address already in use" — Another Streamlit app might be running. Press Ctrl+C in all terminal windows and try again.
